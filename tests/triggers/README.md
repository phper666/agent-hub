# 角色触发验证测试

## 目的

验证每个角色和共享规则的 SKILL.md 的 `description` 字段是否能正确触发。

## 测试方法

对每个角色/规则，设计两类测试用例：

- **Should Trigger（应触发，8-10 条）**：这些 prompt 应该激活该角色/规则
- **Should NOT Trigger（不应触发，8-10 条）**：这些 prompt 不应该激活该角色/规则（应由其他角色响应）
- **边界用例（2-3 条）**：这些 prompt 处于角色边界模糊地带，需人工判断

## 使用方式

1. 在 AI Agent 中加载对应角色
2. 依次输入 Should Trigger 用例，验证角色是否正确激活
3. 依次输入 Should NOT Trigger 用例，验证角色是否未被激活
4. 边界用例用于讨论和优化角色描述边界

## 贡献

新增角色或修改 SKILL.md 的 description 后，请同步更新对应的测试文件。
