# Router 角色触发验证

## Should Trigger（应触发）

| # | Prompt | 预期行为 |
|---|--------|---------|
| 1 | 我应该用哪个角色 | 角色触发，推荐合适的角色 |
| 2 | 切换到PM | 角色触发，切换为PM角色 |
| 3 | 现在是什么角色 | 角色触发，输出当前角色状态 |
| 4 | 帮我看看这个任务应该谁做 | 角色触发，分配合适角色 |
| 5 | 这个文件属于哪个角色的范围 | 角色触发，判断文件归属角色 |
| 6 | 这个需求应该找谁 | 角色触发，推荐处理角色 |
| 7 | 帮我切换到前端工程师 | 角色触发，执行角色切换 |
| 8 | 当前处理角色是什么 | 角色触发，输出当前激活角色 |

## Should NOT Trigger（不应触发）

| # | Prompt | 预期行为 | 应由谁响应 |
|---|--------|---------|-----------|
| 1 | 帮我写一个登录页面 | ❌ 不触发 Router | 应由 Frontend 响应 |
| 2 | 设计数据库表 | ❌ 不触发 Router | 应由 Backend 响应 |
| 3 | 分析用户需求 | ❌ 不触发 Router | 应由 PM 响应 |
| 4 | 这个按钮改个颜色 | ❌ 不触发 Router | 应由 Designer 响应 |
| 5 | 系统架构方案评审 | ❌ 不触发 Router | 应由 Architect 响应 |
| 6 | 写测试用例 | ❌ 不触发 Router | 应由 QA 响应 |
| 7 | 代码审查规范是什么 | ❌ 不触发 Router | 应由 code-review-rules 规则响应 |

## 边界用例

| # | Prompt | 可能的响应角色 | 理由 |
|---|--------|--------------|------|
| 1 | 你能帮我做些什么 | Router 或 任意角色 | 泛化询问可能导致Router角色或其他角色响应 |
| 2 | 团队现在有哪些人 | Router 或 Delivery Director | 团队信息Router可告知但交付总监统筹团队 |
| 3 | 这个任务你来做 | Router 或 当前角色 | 命令式指派可能跳过路由器直接激活角色 |
