# Designer 角色触发验证

## Should Trigger（应触发）

| # | Prompt | 预期行为 |
|---|--------|---------|
| 1 | 设计一个登录页面 | 角色触发，输出页面设计方案 |
| 2 | 这个按钮的配色方案 | 角色触发，提供配色建议 |
| 3 | 画个原型图 | 角色触发，输出原型图 |
| 4 | UI规范应该怎么定 | 角色触发，制定UI规范 |
| 5 | 这个组件布局合理吗 | 角色触发，评审组件布局 |
| 6 | 设计系统的颜色变量怎么定义 | 角色触发，定义设计Token |
| 7 | 移动端适配方案 | 角色触发，输出适配方案 |
| 8 | 交互流程怎么优化 | 角色触发，优化交互流程 |
| 9 | 图标风格统一一下 | 角色触发，统一图标设计 |

## Should NOT Trigger（不应触发）

| # | Prompt | 预期行为 | 应由谁响应 |
|---|--------|---------|-----------|
| 1 | 这个API怎么写 | ❌ 不触发 Designer | 应由 Architect 响应 |
| 2 | 数据库表设计 | ❌ 不触发 Designer | 应由 Backend 响应 |
| 3 | PRD里用户故事完整吗 | ❌ 不触发 Designer | 应由 PM 响应 |
| 4 | React组件逻辑有bug | ❌ 不触发 Designer | 应由 Frontend 响应 |
| 5 | 写单元测试 | ❌ 不触发 Designer | 应由 QA 响应 |
| 6 | 部署配置 | ❌ 不触发 Designer | 应由 Backend(DevOps) 响应 |
| 7 | Git提交规范 | ❌ 不触发 Designer | 应由 git-rules 规则响应 |

## 边界用例

| # | Prompt | 可能的响应角色 | 理由 |
|---|--------|--------------|------|
| 1 | 这个界面交互性能太差了 | Designer 或 Frontend | 涉及交互设计(Designer)与性能实现(Frontend) |
| 2 | 设计稿和实现差距大 | Designer 或 Frontend | 既涉及设计标准又涉及前端实现 |
| 3 | 无障碍访问怎么实现 | Designer 或 Frontend | 涉及设计层面无障碍规范和前端实现 |
