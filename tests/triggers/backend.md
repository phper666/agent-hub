# Backend 角色触发验证

## Should Trigger（应触发）

| # | Prompt | 预期行为 |
|---|--------|---------|
| 1 | 写一个API接口 | 角色触发，输出API接口代码 |
| 2 | 数据库表设计 | 角色触发，输出数据库表设计 |
| 3 | 认证鉴权怎么实现 | 角色触发，设计认证鉴权方案 |
| 4 | SQL优化 | 角色触发，优化SQL查询 |
| 5 | 写个中间件 | 角色触发，输出中间件代码 |
| 6 | 接口文档怎么写 | 角色触发，编写接口文档 |
| 7 | 数据库迁移脚本 | 角色触发，输出迁移脚本 |
| 8 | 缓存策略怎么设计 | 角色触发，设计缓存方案 |
| 9 | 消息队列怎么用 | 角色触发，提供消息队列方案 |

## Should NOT Trigger（不应触发）

| # | Prompt | 预期行为 | 应由谁响应 |
|---|--------|---------|-----------|
| 1 | 写React组件 | ❌ 不触发 Backend | 应由 Frontend 响应 |
| 2 | 画原型图 | ❌ 不触发 Backend | 应由 Designer 响应 |
| 3 | 写CSS动画 | ❌ 不触发 Backend | 应由 Frontend 响应 |
| 4 | 产品需求分析 | ❌ 不触发 Backend | 应由 PM 响应 |
| 5 | 写E2E测试 | ❌ 不触发 Backend | 应由 QA 响应 |
| 6 | 部署监控告警配置 | ❌ 不触发 Backend | 应由 Backend(DevOps) 响应 |
| 7 | 代码审查规范 | ❌ 不触发 Backend | 应由 code-review-rules 规则响应 |

## 边界用例

| # | Prompt | 可能的响应角色 | 理由 |
|---|--------|--------------|------|
| 1 | 这个接口应该用REST还是GraphQL | Backend 或 Architect | 实现细节vs架构决策 |
| 2 | 数据库读写分离怎么搞 | Backend 或 Architect | 具体实现vs架构方案 |
| 3 | 这个API的响应时间太长了 | Backend 或 QA | 性能优化vs性能测试排查 |
