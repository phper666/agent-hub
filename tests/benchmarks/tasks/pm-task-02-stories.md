# PM Task 02: User Stories — 用户认证系统

## 角色
PM (产品经理)

## 输入 Prompt

```
你是一个产品经理。基于以下简要需求，编写详细的用户故事：

## 需求
用户认证系统，支持：
- 邮箱注册
- 邮箱 + 密码登录
- 第三方登录（Google、GitHub）
- 忘记密码 / 重置密码
- 二次验证（可选）

## 要求
- 每个功能 2-3 个用户故事
- 格式：作为 [角色]，我希望 [功能]，以便 [价值]
- 每个用户故事包含 3-5 条验收标准（Given/When/Then）
- 按优先级排序（P0/P1/P2）
- 输出到 docs/current/requirements/user-stories.md
```

## 评估重点
- 用户故事覆盖度（5 个功能各 2-3 个故事）
- 验收标准的具体性
- 优先级划分的合理性
- 异常流程覆盖（邮箱已注册、验证码过期等）
