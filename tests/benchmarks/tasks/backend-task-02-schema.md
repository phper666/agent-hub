# Backend Task 02: Database Schema — 订单系统

## 角色
Backend (后端工程师)

## 输入 Prompt

```
你是一个后端工程师。为电商平台的订单系统设计数据库 Schema：

## 业务需求
- 用户下订单（包含多个商品、数量、单价）
- 订单状态流转：待支付 → 已支付 → 已发货 → 已完成 / 已取消
- 支持优惠券（满减、折扣）
- 支持多个收货地址
- 需要记录支付流水

## 要求
- 写出完整的 DDL（CREATE TABLE 语句）
- 包含：字段类型、主键、外键、索引、约束
- 考虑性能：哪些字段需要索引
- 考虑数据一致性：事务边界
- 输出到 docs/current/database/schema.sql
```

## 评估重点
- 表设计规范性（范式化程度）
- 索引设计合理性
- 外键约束完整性
- 枚举字段的定义
