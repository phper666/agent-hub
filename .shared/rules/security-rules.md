## Security Rules（所有角色的安全底线）

### 输入验证

1. 所有用户输入必须在 API 边界处验证和清洗（包括长度、类型、格式）
2. 参数化查询——禁止拼接 SQL / NoSQL 查询字符串
3. 防止 XSS：输出到 HTML 时必须转义，使用框架自带的自动转义
4. 文件上传必须验证 MIME 类型、扩展名、文件大小（≤ 10MB），存储路径不可预测

### 认证与授权

5. 密码使用 bcrypt 哈希（cost ≥ 12），禁止 MD5 / SHA1
6. JWT 过期时间 ≤ 1 小时，refresh token ≤ 30 天
7. 登录接口限制 5 次/分钟/用户，失败后锁定 15 分钟
8. 敏感操作（改密码、删账号、资金操作）必须二次验证
9. RBAC 权限校验必须在服务端执行，不信任前端传入的角色信息
10. API 端点必须鉴权——默认 deny，显式 allow

### 数据保护

11. 日志禁止记录 PII（手机号、邮箱、密码、Token、身份证）
12. 密钥放在环境变量或密钥管理服务，禁止写入代码或配置文件
13. 数据库中的敏感字段必须加密存储（如银行卡号、身份证号）
14. API 响应不返回多余字段（最小权限原则）
15. 错误响应不返回堆栈信息、SQL 语句或内部路径

### 传输安全

16. 生产环境强制 HTTPS（HSTS header）
17. CORS 只允许白名单域名，不使用 `*`
18. 所有 state-changing 操作需要 CSRF 保护
19. Cookie 设置 `Secure`、`HttpOnly`、`SameSite=Strict`

### 依赖与基础设施

20. 依赖必须定期审计漏洞（`npm audit` / `pip audit` / `go mod verify`）
21. 不使用已知有高危漏洞的依赖版本
22. 生产环境不暴露调试端口、管理面板或 swagger 文档

### 事件响应

23. 安全事件（泄露、入侵）必须在 24 小时内通知相关方
24. 发现漏洞后立即创建 hotfix 分支修复，不等到下一个迭代
25. 定期轮换密钥和 Token（至少每 90 天）
